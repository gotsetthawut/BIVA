# AI Governance

## Purpose

This document defines baseline governance expectations for AI-powered ideas evaluated by BIVA.

## Governance Principles

- Make AI use visible when it affects user decisions.
- Keep humans accountable for high-impact outcomes.
- Protect personal, sensitive, and regulated data.
- Validate output quality before customer-facing use.
- Document risks, controls, owners, and review triggers.
- Avoid automation where errors could create serious harm without review.

## Required Review Triggers

Run an AI risk review when an idea involves:

- Personal data.
- Automated recommendations.
- Regulated industries.
- Financial, legal, health, education, hiring, or safety decisions.
- Customer-facing generated content.
- Data scraped from third-party sources.
- Autonomous agents that take actions for users.

## Minimum Controls

| Control | Purpose |
| --- | --- |
| Human review | Prevent unsupported automated decisions |
| Source tracking | Preserve evidence and auditability |
| Data minimization | Reduce privacy and security exposure |
| Output evaluation | Test accuracy, harmful output, and failure modes |
| Escalation path | Handle disputes, corrections, and edge cases |

## BIVA Rule

An AI idea with high or critical residual risk should receive HOLD or NO-GO until it is redesigned or reviewed by qualified experts.
