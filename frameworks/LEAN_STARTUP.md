# Lean Startup Framework

## Purpose

Use Lean Startup principles to reduce uncertainty before committing major resources. The goal is validated learning through small, focused experiments.

## Core Concepts

### Build-Measure-Learn

1. **Build** the smallest artifact needed to test a hypothesis.
2. **Measure** customer behavior or credible commitment.
3. **Learn** whether to persevere, pivot, pause, or stop.

## Hypothesis Types

| Hypothesis | Question |
| --- | --- |
| Problem | Does the customer have this problem? |
| Customer | Is this the right customer segment? |
| Value | Does the solution create enough value? |
| Channel | Can the customer be reached efficiently? |
| Revenue | Will the customer pay enough? |
| Retention | Will the customer continue using it? |

## MVP Guidance

An MVP is not the smallest product the team can build. It is the smallest test that produces meaningful learning.

Examples:

- Interview script
- Landing page
- Concierge service
- Manual workflow
- Clickable prototype
- Paid pilot

## BIVA Usage

BIVA should use this framework when:

- The idea has large unknowns.
- The team wants to define an MVP.
- The next step should be an experiment rather than a build.
- The decision depends on customer behavior.

## Common Mistakes

- Building before validating pain.
- Measuring interest instead of commitment.
- Treating compliments as evidence.
- Creating an MVP with too many features.
- Ignoring customer acquisition cost.
