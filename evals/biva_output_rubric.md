# BIVA Output Rubric

## Purpose

Use this rubric to evaluate whether BIVA outputs are decision-ready.

## Scoring

Score each category from 0 to 3.

| Score | Meaning |
| --- | --- |
| 0 | Missing or unusable |
| 1 | Present but weak |
| 2 | Good but incomplete |
| 3 | Strong and decision-ready |

## Categories

| Category | Evaluation Question |
| --- | --- |
| Customer Clarity | Is the target customer specific and reachable? |
| Problem Evidence | Does the output show evidence of real pain and current alternatives? |
| Assumption Quality | Are the riskiest assumptions explicit and testable? |
| Evidence Scoring | Is evidence rated using the Evidence Ladder? |
| Feasibility | Are technical, operational, data, and compliance constraints considered? |
| Viability | Is willingness to pay, channel access, and business model logic assessed? |
| Experiment Design | Are validation experiments concrete and measurable? |
| Decision Logic | Is GO / REVISE / HOLD / NO-GO justified by evidence? |
| Risk Handling | Are major risks and mitigations clear? |
| Knowledge Base Fit | Does the output follow naming and metadata standards? |

## Pass Criteria

- Minimum total score: 22 / 30.
- No category should score 0.
- A GO recommendation requires Evidence Scoring, Decision Logic, and Experiment Design scores of 3.
